# NexusNet Teachers: Historical Canon vs Live v2026 Layer

`LOCKED CANON`

## Why Two Registries Exist
NexusNet’s teacher system is not a flat model registry.

It preserves two distinct layers:

- `historical`
  The canonical bootstrapping record. This keeps the original central mentor ensemble, the historical expert-role teacher ensembles, and the original best-ensemble-per-role principle intact.
- `v2026_live`
  The operational layer used for current routing. This adds primary/secondary teacher pairs, bounded LFM2 efficiency coaching, local-first routing preferences, and current teacher pools without rewriting the historical record.

Historical canon is preserved because NexusNet is a brain-first learning system. Teachers are mentors used for bootstrapping, critique, contrast, and internalization. The live layer exists because current execution needs an operational routing map.

## Why Historical Best-Ensemble-Per-Role Is Preserved
The historical layer keeps:

- `best_ensemble_per_role: true`
- `one_teacher_per_role: false`
- `teachers_discarded_after_surpass: true`

This matters because the original system did not assume one permanent teacher for each expert capsule. NexusNet studies ensembles, records disagreements, learns from them, and eventually replaces them when internal experts surpass them.

The live pair map does not erase that. It is an operational simplification for current routing, not a rewrite of the historical training philosophy.

## Why Live Primary/Secondary Pairs Exist
The v2026 layer adds:

- a default primary teacher per expert
- a secondary comparison teacher
- Critique Expert as arbiter
- optional LFM2 bounded efficiency coaching where explicitly allowed
- local-first or mixed-local-then-remote routing preferences

This gives the live system predictable routing while still preserving disagreement capture, arbitration hooks, and later teacher retirement.

## Why Critique Expert Arbitrates
Critique Expert remains the skeptical examiner and safety gate during teacher disagreement.

Its purpose is not to act like a universal master teacher. Its purpose is to:

- arbitrate disagreement traces
- judge promotion evidence
- challenge weak synthesis
- keep safety and rollback discipline visible

## Why LFM2 Is Bounded
LFM2 is preserved as a bounded specialist efficiency coach.

It is allowed in lanes such as:

- router fastpath
- toolsmith work
- memory budgeting
- security tool protocol
- edge-constrained execution
- long-context efficiency support

It is not allowed to become a universal teacher or override correctness and safety in high-risk work.

## Why Teachers Are Versioned
Teacher registries are versioned because NexusNet is not a static wrapper over whatever model is current.

It needs:

- historical continuity
- operational live routing
- provenance on which teacher set was used
- retirement/replacement shadow scaffolding
- compatibility with Dream Training, Federated Continuous Learning, and foundry-native takeover paths

## AOs Are Not Frozen
Assistant Orchestrators remain open-ended.

This teacher implementation does not freeze AOs to a tiny list. It only gives the teacher system the routing, provenance, arbitration, and retirement scaffolding it needs inside the broader NexusNet brain.

## Curriculum Architect Stays Auxiliary
The authoritative roster remains the core 19 expert capsules.

`Curriculum Architect` is preserved as an auxiliary path with canon status:

- `STRONG ACCEPTED DIRECTION`

It does not renumber or overwrite the authoritative 19-core roster unless canon changes explicitly.
