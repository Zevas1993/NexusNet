# Skill Evolution Governance

## Canon
- Skill evolution stays off the hot path unless promoted.
- No uncontrolled live mutation of active skills.
- No raw-user-data promotion into shared skills.

## Operational Additions
- Tool-sequence-aware trajectory clustering
- Shadow proposal artifact generation
- Governed repository storage for proposals
- Refinement outputs with review, promotion, and rollback requirements

## Governance Requirements
- EvalsAO review
- promotion/review gate
- rollback-ready state
- privacy-preserving aggregation
